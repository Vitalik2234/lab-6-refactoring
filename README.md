# Library System — Lab 5  
## Бізнес-логіка, сервіси, модульне тестування

Цей проєкт реалізує архітектуру **Controller → Service → Repository** та моделює роботу
бібліотеки. У застосунку використано **DTO**, **Google‑style docstrings**, **flake8**
та **pytest** для модульного тестування і забезпечення якості коду.

##  Реалізовані бізнес‑сценарії

### 1) Видача книги
- Перевірка існування книги  
- Перевірка існування користувача  
- Перевірка, чи книга не видана  
- Позначення книги як виданої (is_issued = True)

### 2) Повернення книги
- Перевірка, що книга існує  
- Перевірка, що книга була видана  
- Скидання статусу (is_issued = False)

### 3) Пошук книги за назвою
- Точне співпадіння назви  
- Повертається список знайдених книг

### 4) Реєстрація користувача
- Додавання користувача в in‑memory репозиторій
